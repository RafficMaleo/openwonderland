Each Cell in the world has a bounds defined by the developer. The bounds must encapsulate all the geometry for that cell, if the geometry moves then it may be necessary to update the bounds of the cell. The developer is responsible for ensuring that the bounds always encapsulates the geometry. Cell bounds are defined in Cell local coordinates.

The bounds of a cell can overlap the bounds of it's sibling cells.

getComputedWorldBounds returns a bounds object that encapsulates this cell and all it's child cell bounds. The computedWorldBounds is only updated periodically so there may be some lag in these bounds encapsulating children that have changing bounds. The timing of updates is not strongly defined at this time in order to allow exploration of various optimisations in the future.
